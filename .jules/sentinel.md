## 2024-05-14 - Path Traversal in File Deletion Router
**Vulnerability:** Found a Path Traversal vulnerability in the `DELETE /skills/:name` endpoint where `req.params.name` was appended to a directory path (`join(skillsDir, req.params.name)`) without any sanitization. This allowed the deletion of arbitrary files across the system using `../` (e.g., `DELETE /skills/..%2F..%2Fetc%2Fpasswd`).
**Learning:** This existed because while `POST /skills/upload` sanitized the uploaded name with `basename()`, the `DELETE` endpoint failed to repeat this validation, assuming route parameters from an API request to be inherently safe.
**Prevention:** Always validate and sanitize dynamic route parameters that interact directly with the filesystem, typically by using `basename()` or enforcing regex match on safe characters (`^[a-zA-Z0-9_-]+$`). Do not assume parameters originating from a routing framework are sanitized or inherently safe to be concatenated directly into filesystem paths.
